# AI policy

AI tools are welcome on this project. What makes a change acceptable is not *how*
it was written -- it is that a human has read every line and stands behind it.
The bar is ownership, not authorship.

## Own what you submit

Use whatever tools you like, AI included. The one rule: a human reads every line
and vouches for it as their own work.

- A pull request marked **Ready for review** is your assertion that you have read
  every line and stand behind it. Only ready PRs are reviewed for merge.
- **Draft pull requests are welcome** for AI-assisted first passes, experiments,
  and work in progress. A draft is a conversation, not a submission; it will not
  be merged, and that is fine.

Do not mark a PR ready until you have reviewed it as if you had typed it by hand.

Issues follow the same rule: describe the problem in your own words. Issues that
are clearly unreviewed AI output -- generic, padded, or inaccurate about the
project -- will be closed. Not because a tool was used, but because no human
stood behind them.

## How the maintainer uses AI

For transparency: I use AI heavily -- to draft code, open issues, and write the
first version of pull requests. Every line is reviewed and signed off before it
lands, and by merging I take ownership of it. The difference from a contributor
is only *when* the human review happens: on my own PRs I review at merge time;
contributors review before marking a PR ready. The standard is the same for
everyone -- a human owns every merged line.

Media (sounds, images, videos) is not AI-generated.

## A living document

How I use AI, where review happens, and what I ask of contributors are still
being worked out as the project grows, so this policy will change. If something
here is unclear or seems wrong, open an issue and say so.

The contribution mechanics live in [`CONTRIBUTING.md`](CONTRIBUTING.md).
